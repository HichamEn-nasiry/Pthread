Fonctions pthreads en C

pthread_create

int pthread_create(pthread_t *thread, const pthread_attr_t *attr, void *(*start_routine)(void *), void *arg)

Cette fonction crée un nouveau thread. Elle prend comme arguments un pointeur vers un identificateur de thread (pthread_t *), des attributs du thread (pthread_attr_t *), une fonction qui sera exécutée par le thread (void *(*start_routine)(void *)), et un argument optionnel qui sera passé à la fonction du thread (void *arg).

pthread_join

int pthread_join(pthread_t thread, void **retval)

Cette fonction suspend l'exécution du thread appelant jusqu'à ce que le thread spécifié par thread se termine. Elle renvoie la valeur de sortie du thread dans retval, qui peut être récupérée par le thread appelant.

pthread_exit

void pthread_exit(void *retval)

Cette fonction termine l'exécution du thread en cours et renvoie une valeur de sortie (retval) facultative. La valeur de sortie peut être récupérée par d'autres threads en utilisant la fonction pthread_join.

pthread_mutex_t

typedef struct pthread_mutex_t pthread_mutex_t;

Le type pthread_mutex_t représente un mutex (verrou) utilisé pour la synchronisation entre les threads. Un mutex permet d'éviter les problèmes de concurrence lorsqu'un certain nombre de threads tentent d'accéder simultanément à une ressource partagée.

pthread_mutex_lock

int pthread_mutex_lock(pthread_mutex_t *mutex)

Cette fonction acquiert un verrou mutex spécifié par mutex. Si le mutex est déjà verrouillé par un autre thread, le thread en cours sera mis en attente jusqu'à ce que le verrou soit disponible.

pthread_mutex_unlock

int pthread_mutex_unlock(pthread_mutex_t *mutex)

Cette fonction libère un verrou mutex spécifié par mutex. Si d'autres threads attendent le verrou, l'un d'entre eux sera choisi pour acquérir le verrou.

pthread_barrier_t

typedef struct pthread_barrier_t pthread_barrier_t;

Le type pthread_barrier_t représente une barrière de synchronisation utilisée pour bloquer l'exécution des threads jusqu'à ce qu'un certain nombre de threads atteignent la barrière.

pthread_barrier_init

int pthread_barrier_init(pthread_barrier_t *barrier, const pthread_barrierattr_t *attr, unsigned int count)

Cette fonction initialise une barrière de synchronisation spécifiée par barrier. Elle prend également un attribut de barrière optionnel (pthread_barrierattr_t *) et le nombre de threads nécessaires pour dépasser la barrière (count).

pthread_barrier_wait

int pthread_barrier_wait(pthread_barrier_t *barrier)

Cette fonction bloque l'exécution du thread appelant jusqu'à ce que le nombre requis de threads atteigne la barrière. Lorsque le nombre requis de threads est atteint, tous les threads sont libérés simultanément.

pthread_cond_t

typedef struct pthread_cond_t pthread_cond_t;

Le type pthread_cond_t représente une variable de condition utilisée pour la synchronisation entre les threads. Une variable de condition permet à un thread de suspendre son exécution jusqu'à ce qu'une condition particulière soit satisfaite.

pthread_cond_wait

int pthread_cond_wait(pthread_cond_t *cond, pthread_mutex_t *mutex)

Cette fonction bloque l'exécution du thread appelant jusqu'à ce que la variable de condition spécifiée par cond soit signalée par un autre thread. L'appel à cette fonction nécessite un verrou mutex spécifié par mutex, qui sera temporairement relâché pendant l'attente.

pthread_cond_signal

int pthread_cond_signal(pthread_cond_t *cond)

Cette fonction envoie un signal à la variable de condition spécifiée par cond. Si plusieurs threads attendent sur cette variable de condition, un seul thread est choisi pour être réveillé.
